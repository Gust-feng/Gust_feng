# Linux的文件

## 文件命名

> 优秀的命名方式是管理文件的最佳实践

Linux文件名的最大长度是255字符，通常由字母、数字、“.”、“_”、“-”字符组成。通常以“.”开头的文件名是隐含文件，即在文件列表中不显示（并不代表不存在）

与Windows不同的是，Linux是严格区分大小写的，大小写不同的文件会被认为是两个不同的文件

## 通配符

通配符并不表示某一个具体的字符，而是代表多种选择

* `?`：表示匹配任意的**单个**字符

* `*`：匹配0或**多个任意**字符，隐含文件除外

* `[]`：匹配方括号中列出的字符集中的任何**单个**字符

   * 列举：逐个列出各个字符，如[ABC]表示由A、B、C三个字符构成的字符集合

   * 范围：用“-”描述字符范围，如[a-z]表示由所有小写字母构成的集合。需要注意，范围内可指定多个范围，如[a-z A-Z]，但需要按照升序的方式排列

     > | 模式        | 匹配内容                       |
     > | :---------- | :----------------------------- |
     > | `[a-zA-Z]`  | 所有大小写字母（a-z 和 A-Z）   |
     > | `[a-z A-Z]` | 小写字母 **+ 空格** + 大写字母 |

   * 排除：使用“!”排除字符，如[!A-Z]表示除大写字母以外的所有字符构成的集合

当命令的参数中出现通配符时，Shell并不把参数直接传递给命令，而是把它看作一个文件名模式的字符串。

> ### 举个栗子：
>
> 1. 执行流程
>
> ```shell
> ls *.txt
> ```
>
> 实际发生的事情：
>
> ```shell
> 1. 用户输入: ls *.txt
>          ↓
> 2. Shell 扫描当前目录，找到匹配的 *.txt 文件
>    → file1.txt file2.txt file3.txt
>          ↓
> 3. Shell 把展开后的列表传给 ls 命令
>    → 实际执行: ls file1.txt file2.txt file3.txt
>          ↓
> 4. ls 命令接收到的参数是具体的文件名，不是 "*.txt"
> ```

因此，当命令执行时得到的实际参数时所有匹配的文件名的序列

## 文件类型

Linux 文件主要分为 **7 种类型**：

| 符号 | 类型         | 说明                   | 示例                      |
| :--- | :----------- | :--------------------- | :------------------------ |
| `-`  | **普通文件** | 文本、二进制、数据     | `file.txt`, `a.out`       |
| `d`  | **目录**     | 存放其他文件的容器     | `/home`, `/etc`           |
| `l`  | **符号链接** | 指向其他文件的快捷方式 | `link -> /usr/bin/python` |
| `b`  | **块设备**   | 按块读写（磁盘等）     | `/dev/sda`, `/dev/nvme0`  |
| `c`  | **字符设备** | 按字符流读写           | `/dev/tty`, `/dev/null`   |
| `s`  | **套接字**   | 进程间网络通信         | `/var/run/docker.sock`    |
| `p`  | **管道**     | 进程间单向通信（FIFO） | 临时创建的命名管道        |

> ### 举个栗子
>
> ```shell
> ls -la
> ```
>
> 输出：
>
> ```shell
> -rw-r--r-- 1 user group  1234 Mar 04 10:00 file.txt      # - 普通文件
> drwxr-xr-x 2 user group  4096 Mar 04 10:01 mydir/        # d 目录
> lrwxrwxrwx 1 user group     9 Mar 04 10:02 link -> file  # l 软链接
> crw-rw-rw- 1 root root   1, 3 Mar 04 09:00 null         # c 字符设备
> brw-rw---- 1 root disk   8, 0 Mar 04 09:00 sda          # b 块设备
> srwxr-xr-x 1 user group     0 Mar 04 10:03 docker.sock  # s 套接字
> prw-r--r-- 1 user group     0 Mar 04 10:04 mypipe       # p 命名管道
> ```

## 文件所属关系

**每个文件属于一个用户（owner）和一个组（group）**，这是 Linux 权限控制的基础。

## 三层归属关系

```shell
-rwxr-xr-- 1 owner group 1234 Mar 04 file.txt
 |   |   |
 |   |   └── others: r-- (只读)
 |   └────── group:  r-x (读+执行)
 └────────── owner:  rwx (读+写+执行)
```



| 维度               | 字段    | 含义                               |
| :----------------- | :------ | :--------------------------------- |
| **用户（User）**   | `owner` | 文件创建者，拥有最高权限           |
| **组（Group）**    | `group` | 文件所属的用户组                   |
| **其他（Others）** | -       | 系统中除 owner 和 group 外的所有人 |

## 文件访问权限

* **文件的访问权限**

  **Linux 文件权限 = 读(r) + 写(w) + 执行(x)**

| 权限      | 文件     | 目录                 |
| :-------- | :------- | :------------------- |
| **r** (4) | 读取内容 | 列出目录内容（`ls`） |
| **w** (2) | 修改内容 | 创建/删除/重命名文件 |
| **x** (1) | 执行程序 | 进入目录（`cd`）     |
| -         | 无权限   | 无法违法操作         |

* **文件权限的分配范围**

  一个文件可能会被多个用户使用，因此需要设置文件权限以满足不同用户的不同需求和权力

​	Linux将每个文件的用户分为属主（user）、组用户（group)、和其他人（other）3类

> ROOT用户不受访问权限限制

* 文件类型与权限表示法

  **完整表示格式：`drwxrwxrwx`**

  ```shell
  文件类型 | owner权限 | group权限 | others权限
     d    |   rwx    |   rwx    |   rwx
   			|   	   |         |
   			|   	   |         └── others: 其他用户权限
   			|   	   └────── group:  所属组权限  
   			└────────── owner:  文件所有者权限
  ```

  

  | 权限 | 数值 | 文件作用 | 目录作用      |
  | :--- | :--- | :------- | :------------ |
  | `r`  | 4    | 读取内容 | 列出目录内容  |
  | `w`  | 2    | 修改内容 | 创建/删除文件 |
  | `x`  | 1    | 执行程序 | 进入目录      |

**常用组合**

```shell
-rw-r--r--  (644)  普通文件，owner可写，其他只读
drwxr-xr-x  (755)  目录，所有人可进入
-rwxr-x---  (750)  程序，owner和group可执行，其他无权限
lrwxrwxrwx  (777)  软链接（链接本身权限无意义）
```
